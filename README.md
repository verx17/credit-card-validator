# Credit Card Validator

## 📋 Описание проекта

Консольная утилита на языке Go для быстрой валидации номеров банковских карт и определения банка-эмитента.

Приложение выполняет три этапа проверки:
1. Длина номера (13-19 символов) и допустимые символы (только цифры).
2. Проверка корректности номера по [**алгоритму Луна**](https://ru.wikipedia.org/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%9B%D1%83%D0%BD%D0%B0).
3. Определение банка по первым 6 цифрам карты (BIN) на основе базы данных.

## 🛠 Основные функции

- **Гибкая загрузка данных:** База банков подгружается из внешнего CSV-файла при старте. Путь к файлу можно задать через флаг `-db`.
- **Безопасный парсинг:** Встроенная защита от некорректно заполненных строк в файле базы данных.
- **Интерактивный ввод:** Бесконечный цикл обработки пользовательских запросов.
- **Оптимизированная очистка:** Программа за один проход `O(n)` удаляет пробелы и дефисы из введенного номера без лишних аллокаций памяти.
- **Строгая валидация:** Проверка на наличие посторонних букв/символов и соответствие длине стандарта (13-19 цифр).
- **Алгоритм Луна:** Реализация стандартной проверки контрольной суммы для защиты от опечаток.
- **Быстрый поиск по BIN:** Определение банка с использованием бинарного поиска по диапазонам BIN.

## 🚀 Установка и запуск

### Предварительные требования
- Установленный **Go** (версии 1.22 или выше).
### Установка

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/verx17/credit-card-validator.git
   ```
2. **Перейдите в директорию проекта**
   ```bash
   cd credit-card-validator
   ```
3. **Подготовка файлов**
Создайте файл данных `banks.txt` в одной директории с программой. Формат файла: `Название Банка,BIN_Начало,BIN_Конец`

**Пример `banks.txt`:**
```
Lunar Bank,400000,499999
Mars Credit Union,500000,599999
Venus Express Bank,600000,699999
```
### Запуск программы
Откройте терминал в папке с проектом и выполните команду:
```bash
go run main.go
```
### Запуск программы с флагом -db
Если ваш файл называется иначе или лежит в другой папке, используйте флаг `-db`:
```bash
go run main.go -db=my_custom_banks.csv
```
Или скомпилируйте бинарный файл:
```bash
go build -o Validator
./Validator -db=banks.txt
```

## Примеры использования

**Сценарий 1: Успешная валидация**

Пользователь вводит корректный номер карты, который есть в базе (в данном примере BIN 555544 попадает в Mars Credit Union).
```Plaintext
Загрузка программы валидации банковских карт космической системы...
Введите номер карты (или пустую строку для выхода): 5555 4444 3333 2222
Карта 5555444433332222 относится к банку: Mars Credit Union
```

**Сценарий 2: Ошибка алгоритма Луна**

Пользователь ошибся в одной цифре (контрольная сумма не совпала)
```Plaintext
Введите номер карты (или пустую строку для выхода): 4000001234567890
Ошибка: введённый номер карты не прошёл проверку по алгоритму Луна
```

**Сценарий 3: Банк не найден**

Номер карты валиден по алгоритму Луна, но его BIN отсутствует в файле banks.txt.
```Plaintext
Введите номер карты (или пустую строку для выхода): 1111222233334444
Внимание: не удалось определить банк для карты 1111222233334444
```

**Сценарий 4: Неверный формат**

Пользователь ввел недопустимые символы или слишком короткий номер.
```Plaintext
Введите номер карты (или пустую строку для выхода): 4000 abc
Ошибка: введён некорректный формат номера карты (допускаются только цифры, пробелы и тире, длина от 13 до 19 цифр)
```

**Сценарий 5: Выход из программы**

Нажатие клавиши `<Enter>` с пустой строкой (или `<Ctrl D>`) корректно завершает работу.
```Plaintext
Введите номер карты (или пустую строку для выхода): 
>
Ввод завершён. Закрытие программы.
```
